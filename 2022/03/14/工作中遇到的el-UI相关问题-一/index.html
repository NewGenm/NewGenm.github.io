工作中遇到的el-UI相关问题(一)

发表于 2022-03-14 更新于 2022-03-20 阅读次数：

记录工作上遇到el-UI相关的问题
el-form、el-scrollbar、el-select、el-date-picker、el-table

el-form相关

el-form最好按文档的结构来写

rules相关

rules的里规定的值一定要在form的:model中，
即

name要在abc中

prop相关

深度检查，如果所要检查的属性不在 form 的 :model的第一层中

需要修改prop中的名称，添加一层父组件

关于el-scrollbar

重点：包裹scrollbar的容器一定要大过scrollbar，要不然会有一部分显示不出来！！！！

关于el-select（选择）

使用el-select的时候，选项el-option的value值是布尔类型的时候，需要给value绑定动态，即:value，要不然不会显示el-option对应的label值

关于el-date-picker（日期选择器）

修饰value的格式，里面必须全大写

关于el-table

前端控制分页

State.goodsFilter.silce() 对数据进行拆分
array.slice(start, end) 数组的拆分方法 start 开始的位置 end 结束的位置
（currentPage – 1）* size 当前所在页面-1 * 一页页面量（如果是第一页就是 （1-1）size = 0）
currentPage * size 当前所在页面 一页数据量
开始到结尾的数据量正好是一页的数据量

后端控制分页

如果分页是后端处理，即一次请求只请求一页，防止换页后多选消失，在el-table上添加 :row-key


在多选的el-table-column上添加 :reserve-selection=”true”


修改行相关样式

:row-style=”{height: ‘0px’}”
:cell-style=”{padding: ‘0’}”


重点：table表单的 :data设置默认值的时候一定是数组 [ ] ！！！

el-date-picker

修饰value的格式，里面必须全大写
